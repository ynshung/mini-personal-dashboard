# Idle Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enter a sleep screen after 10 minutes of consecutive server unreachability, stopping all polling until any button wakes the device.

**Architecture:** All changes are confined to `src/main.cpp`. A new `IDLE_TIMEOUT_MS` constant drives the timeout. A single `serverUnreachableSince` timestamp (shared across both pollers) measures consecutive failure duration. A new `IDLE` screen state stops polling naturally (the existing `if (activeScreen == SPOTIFY)` / `else if (activeScreen == CC_USAGE)` blocks skip idle). Button callbacks check for idle at entry and call `wakeFromIdle()` instead of their normal action.

**Tech Stack:** C++ / Arduino / PlatformIO / ESP32. No test framework available — verification is via `pio run` (build) + serial monitor observation.

### Task 5: Add `wakeFromIdle()`

**Files:**
- Modify: `src/main.cpp` — insert before `setup()`

`wakeFromIdle()` must be defined after both fetch functions since it calls them directly. Place it immediately before `void setup()`.

- [ ] **Step 1: Add the function**

Immediately before `void setup() {`, insert:

```cpp
void wakeFromIdle() {
    activeScreen = prevScreen;
    serverUnreachableSince = 0;
    pollFailed = false;
    if (activeScreen == CC_USAGE) {
        ccNeedsFullRedraw = true;
        drawCCUsage();
        fetchCCUsage();
        lastCCPoll = millis();
    } else {
        hasArt = false;
        current.track_id = "\x01";
        drawStatus("Loading...");
        fetchNowPlaying();
        lastPoll = millis();
        lastTick = lastPoll;
    }
}
```
